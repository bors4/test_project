# Инструкция по работе с тестовым фреймворком для onliner.by

## Общее описание

Это проект автоматизированного тестирования веб-интерфейса сайта Onliner.by, созданный на основе WebdriverIO v9 и Cucumber BDD. Проект использует Page Object Model для структурирования тестов и поддерживает различные режимы выполнения тестов.

## Структура проекта

- features/ - Gherkin-фичи с описанием тестовых сценариев
- src/page-objects/ - Реализации Page Object для различных страниц сайта
- src/step-definitions/ - Реализации шагов для фич-файлов
- src/utils/ - Вспомогательные утилиты
- src/config/ - Файлы конфигурации
- .bash/command_menu.bat - Командное меню для быстрого запуска различных команд

## Требования

- Node.js (версия, поддерживаемая зависимостями в package.json)

## Установка

1. Убедитесь, что Node.js установлен
2. Клонируйте репозиторий
3. Выполните установку зависимостей:

```
npm install
```

## Запуск тестов

- npm run test – Запуск всех тестов в headless режиме
- npm run test:debug – Запуск всех тестов с UI для отладки
- npm run test:video – Запуск с записью видео
- npm run test:debug:video – Отладочный запуск с записью видео
- npm run test:feature -- <имя_файла> – Запуск отдельного feature файла
- npm run test:tags -- <тег> – Запуск по тегам (например, @smoke)

## Использование командного меню

Вы можете использовать command_menu.bat в каталоге .bash для удобного выбора нужной команды через интерфейс меню.

## Линтер и форматирование

- npm run lint – Проверка кода линтером
- npm run format – Форматирование кода с помощью Prettier
- npm run lint:write – Исправление ошибок линтера

## Отчеты

- Все тесты генерируют Allure отчеты
- npm run allure:generate – Генерация отчета
- npm run allure:open – Открытие отчета в браузере
- npm run allure:watch – Просмотр отчета в реальном времени

## Конфигурация

- wdio.conf.js – Основной конфигурационный файл WebdriverIO
- .env – Файл с переменными окружения (логины, пароли и т.д.)
